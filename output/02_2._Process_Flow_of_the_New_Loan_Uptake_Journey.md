# 2. Process Flow of the New Loan Uptake Journey

This process flow illustrates the enhanced, data-driven journey for a customer to discover and apply for a loan product through the mobile banking app.

```mermaid
graph TD
    A[Customer Opens Mobile Banking App] --> B{Customer Data Analysis Engine (CDAE) - Real-time}
    B -- Customer Profile & Behavior --> C(Identify Potential Loan Need / Eligibility)
    C --> D{Personalized Loan Offer Generation}
    D -- Relevant Offers & Calls to Action --> E[Display Personalized Offers on App Dashboard / Dedicated Section]

    E --> F{Customer Views Offer}
    F -- Interested --> G[Click on Offer for Details]
    G --> H[View Loan Product Details & Pre-Qualification Info]
    H -- Apply --> I[Initiate Loan Application]
    I --> J[Pre-fill Application with Customer Data]
    J --> K[Customer Reviews & Supplements Information]
    K -- Submit --> L[Application Submitted to Loan Origination System (LOS)]
    L --> M[Automated Underwriting & Approval]
    M -- Approved / Conditional --> N[Notify Customer of Application Status (In-App / Push)]
    N -- Accept Offer --> O[Customer Accepts Loan Offer (e-Sign)]
    O --> P[Loan Disbursed / Account Funded]
    P --> Q[Loan Servicing Begins]

    F -- Not Interested / Dismiss --> R[CDAE Learns from Dismissal]

```
